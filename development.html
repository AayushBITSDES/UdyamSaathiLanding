The Challenge

Querying 30TB+ of unstructured government PDF data with high latency and low accuracy using traditional keyword search results in a fragmented user experience. Micro-enterprises need answers, not document links.

The Solution

A verticalized RAG (Retrieval-Augmented Generation) pipeline optimized for Indian vernacular languages, delivering semantic understanding and voice-first interactions on low-bandwidth networks.

Core Stack

TypeScript + Tailwind

FRONTEND

Type safety for complex state management (voice sessions) and utility-first styling for rapid iteration and performance.

Go (Golang)

BACKEND

High-concurrency handling for real-time voice streams and low-latency API responses. Built for scale.

RAG Pipeline

INTELLIGENCE

Context-aware retrieval that grounds LLM responses in official government notification data, significantly reducing hallucinations.

Cloud Native

INFRASTRUCTURE

Containerized microservices architecture ensuring modularity and scalable deployment across diverse environments.

Why Open Source?

DPI Transparency

We believe Digital Public Infrastructure should be transparent and auditable. Trust is built through code visibility.

Privacy by Design

Voice data is processed ephemerally. We architect systems where user data privacy is a default, not a setting.

Community Driven

Building for India requires diverse inputs. Open source allows the community to contribute localization fixes and domain knowledge.

Why Build in Public?

Accountability

Every commit, architectural decision, and roadmap item is visible. We answer to the community.

Faster Feedback Loops

By sharing early prototypes, we get critical feedback from users and developers before investing months in the wrong direction.

Collaborative Innovation

The challenges of India's linguistic diversity are too big for one team. We invite contributions to solve them together.
